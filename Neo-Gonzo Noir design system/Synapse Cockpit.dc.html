YOU
How does simulated annealing differ from a genetic algorithm for randomized optimization, and when would you pick each?
SYNAPSE
Simulated annealing is a single-solution search: it perturbs one candidate and accepts worse moves with a probability set by a falling temperature schedule, so early exploration gives way to late exploitation1. A genetic algorithm instead evolves a population, recombining high-fitness individuals via crossover and mutation — letting it cover several basins at once2.
Prefer SA when evaluations are cheap and the landscape is rugged but locally smooth; prefer a GA when structure recombines well and you can afford many parallel evaluations3. SA needs almost no memory; a GA trades memory for breadth.

YOU
My A2 simulated-annealing loop converges too fast. Here's my accept rule — what's wrong, and can you give me a clean reference?
CODE BUDDY
You're accepting uphill moves with dE < 0 — that flips the Metropolis test, so worse moves win and the walk never settles. It should accept better moves outright and worse ones only with probability e^(dE/T). Here's a corrected reference grounded in your lecture-07 notes:
sa_reference.py · python copy SAVE TO KB
from math import exp; from random import random
def simulated_annealing(problem, schedule, max_iters=1000):
    state = best = problem.random_state()
    for t in range(max_iters):
        T = schedule(t) # cooling: T → 0
        nxt = problem.neighbor(state)
        dE  = problem.fitness(nxt) - problem.fitness(state)
        if dE > 0 or random() < exp(dE / T): # accept
            state = nxt
        if problem.fitness(state) > problem.fitness(best):
            best = state
    return best

Simulated Annealing vs Genetic Algorithms
## Core difference
SA walks a single solution and accepts worse moves on a falling temperature schedule — exploration early, exploitation late. GA evolves a whole population, recombining good partial solutions via crossover.
## Temperature
The accept probability is the Metropolis rule → exp(-ΔE / T). As T → 0 it collapses to greedy hill-climbing.

CS 7641 · LECTURE 07 — RANDOMIZED OPTIMIZATION
§3 Simulated Annealing
Simulated annealing maintains a single candidate and proposes a local move at each step. A move that improves fitness is always taken; a move that worsens it by ΔE is accepted with probability governed by a temperature parameter T.
P(accept) = exp( −ΔE / T )
High temperatures permit broad exploration; the cooling schedule lowers T over time, concentrating the search. In the limit T → 0, annealing degenerates to greedy hill-climbing and exploits the nearest optimum.
Contrast with population methods (§4), where crossover recombines partial solutions across many candidates at once.
